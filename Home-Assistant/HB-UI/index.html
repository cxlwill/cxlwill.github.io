Homebridge Web 管理界面

发表于 2018-03-17 | 分类于 Home Assistant
HB Web 界面插件最近频繁更新，上线 web 设备管理面板，感觉乔布斯的棺材板要压不住了！


Homebridge 最近伴随着插件 homebridge-config-ui-x 更新到了回春之际。

最新版本的插件除了原有的 HB 概览页面以外竟然加入了设备管理界面，活生生把家庭应用搬到了网页上。为什么说这是创世之举？因为采用基于 Web 的页面就意味着全平台的通用性，现在除了 Siri 语音体验，安卓平台也可以享受到 Homebridge 带来的便利了。

乔布斯：…………

启用该页面的方法也很简单，安装插件之后以 Insecure 模式运行 Homebridge 即可 homebridge -I。大家也不要对这个不安全模式担心，只要不把端口转发到外网，走 Apple 的 iCloud 都是可以放心的。（云上贵州安不安全那我可说不准了……）

那已经将 HB 转为系统服务后台运行的用户怎么办呢？

还不拖到篇末赞赏一下？

编辑homebridge服务文件，我的文件路径是 /etc/systemd/system/homebridge.service，在 ExecStart=/usr/lib/node_modules/homebridge/bin/homebridge $HOMEBRIDGE_OPTS 的最后加一个 -I 即可。当然保存退出后不要忘记重载服务：sudo systemctl daemon-reload。重启 homebridge，web 管理页面就会出现了。

由于尚处于测试功能阶段，管理页面的各方面功能不是非常完善，例如名称的修改、全类型设备的支持等等。但是，不得不说已经是极具突破性的一步了！

现在，没有 iOS 设备的你，一样可以享受 Apple Homekit 了~
